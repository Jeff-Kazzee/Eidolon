## Purpose

SOPs ensure consistency and quality across:

- Development workflows
- Code review processes
- Release management
- Security practices
- Launch preparations

## Using SOPs

1. **Before starting a task**, check if an SOP exists
2. **Follow the checklist** step by step
3. **Update the SOP** if you find improvements
4. **Propose new SOPs** for recurring tasks

## SOP Template

```markdown
# [Procedure Name]

## Overview
Brief description of what this procedure covers.

## When to Use
Situations where this SOP applies.

## Prerequisites
What you need before starting.

## Steps
1. Step one
2. Step two
3. Step three

## Checklist
- [ ] Item 1
- [ ] Item 2

## Troubleshooting
Common issues and solutions.

## Related
Links to related SOPs or documentation.
```
